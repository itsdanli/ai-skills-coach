Day 1 — Quote-First Grounding

Twenty minutes from now you'll be able to force Claude to anchor every claim about a long document to a verbatim quote, so it stops inventing details when you feed it a 40-page transcript or a customer JSON export.

Why this matters

When you paste a long call transcript or a big environment export and ask "what did the customer commit to?", the failure isn't that Claude refuses — it's that it confidently paraphrases something that's almost in the document but not quite. A due date shifts by a week, an owner gets swapped, a "we'll consider it" becomes "they agreed." You don't catch it because the answer reads plausibly, and now a wrong action item is in Linear. The longer the context, the worse the drift: relevant facts get diluted by everything else in the window.

The mechanic

Split the work into two forced phases in one prompt: first Claude extracts verbatim quotes into a scratchpad, then it answers using only those quotes. Quoting first pulls the relevant needles out of the haystack before reasoning starts, and the quotes become an audit trail you can spot-check.

<document>
{{PASTE FULL TRANSCRIPT HERE}}
</document>

First, in <evidence> tags, extract every verbatim quote from the
document that states a commitment, a date, an owner, or a blocker.
Copy the text exactly — do not paraphrase. If none exist, write "none".

Then, in <answer> tags, list the customer's commitments. Every line
must cite one of the quotes above. If a commitment is implied but not
stated in a quote, put it under a separate "Inferred (unverified)"
heading so I can see it's not grounded.

Why each piece is there: the <document> wrapper lets Claude tell "the source" apart from "the instructions." The verbatim constraint plus "do not paraphrase" blocks the drift — a copied quote can't silently mutate a date. Naming the categories (commitment / date / owner / blocker) focuses extraction instead of dumping the whole doc back. And the "Inferred (unverified)" bucket is the key move: it gives Claude a legitimate place to put soft signals, so it stops smuggling them into the grounded list.

The upgrade over the naive version

Before — one-shot ask:

Here's the transcript. What did the customer commit to?

Claude answers in one pass, blending real quotes, inference, and filler with no way to tell which is which. To verify, you re-read the whole transcript — so you've saved nothing.

After — quote-first (the prompt above). You get an <evidence> block of exact quotes and an answer where each item points to one. Verifying means glancing at four or five quotes, not re-reading 40 pages. Anything ungrounded is quarantined under "Inferred," which is exactly the stuff worth a human double-check.

Try it now (10 minutes)

Grab your most recent real customer call transcript or a long email thread — something 1,000+ words where you already know the true answer. Paste it into the prompt above and run it in Claude. A successful output has an <evidence> block where every quote is findable with Ctrl-F in the source (copy one and check), and an <answer> where nothing in the grounded section is missing its citation. The tell that it worked: at least one item you'd have accepted from a one-shot answer now correctly sits under "Inferred (unverified)." That's a wrong action item you didn't create.

Apply it to my workflow

In transcript-to-Linear ticket generation, run the quote-first pass before drafting issues so each ticket carries the exact customer sentence that justifies it — reviewers approve faster and scope creep drops. When auditing a Plenful customer JSON export, ask for verbatim query names and table references before any summary, so the overview never cites an automation that doesn't exist. And in escalation write-ups, quote-first turns "the customer is blocked on X" into a defensible claim with the line that proves it — which matters when eng pushes back.

One-line bookmark

Make Claude quote before it concludes: verbatim evidence first, answer second, inference quarantined.
